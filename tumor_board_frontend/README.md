# Tumor Board (CrisPRO Oncology Arbiter) — Frontend Package

**Regulatory posture:** Research Use Only. Not FDA-cleared, not CE-marked, not for clinical use.
**Data posture:** Both cases are `EXAMPLE-DATA-RUO` fixtures. No real patient data.
**Generated:** 2026-07-19

This directory is a frontend hand-off — HTML mockups, a JSON component spec, live
end-to-end transcripts against the local arbiter, and a machine-checkable acceptance
suite for two real cases. It is not a live frontend. The design system team owns
final visual design, tokens, and framework integration.

## 2. Why two cases

An honest split, driven by what the arbiter actually contributes on each case.

### NSCLC / LUNA16 — end-to-end reasoning
The arbiter runs a MONAI `lung_nodule_ct_detection` RetinaNet bundle over a
16-slice synthetic CT, produces a real detection box + score, feeds it through
a heuristic lung-nodule branching rule, and finishes with an NCCN-lite therapy
map plus an Elo re-ranking of the four candidate actions. Every L1→L5 layer is
computed live against real code. Detector low-confidence (top score 0.042) is
surfaced inline as a honest signal that a 41.5 mm mass is out-of-distribution
for LUNA16 (trained on 3–30 mm nodules); the heuristic still produces a HIGH
risk bucket via the mass-branch threshold.

### AK MBD4-LOF — manuscript-anchored bundle ingestion + audit
All p-values, effect sizes, n_mut / n_wt counts, and the PARP falsification
result come from a SHA-locked `crispro-backend-v2 @ bfd6d11f…` bundle keyed to
manuscript repo SHA `d33f6403…`. **The arbiter did not generate them.** On
this case the arbiter contributes:

1. Contract validation against the `TumorBoardBundle` pydantic schema.
2. A URL honesty gate that drops non-`seed_urls` references from any
   co-scientist reflect pass.
3. The `bundle_sha256` audit-of-record echoed back on
   `POST /v1/tumor_board/bundle`.
4. A deterministic Elo audit sort with the manuscript-derived weights
   surfaced as labeled modifiers (`manuscript_lead_axis_boost`,
   `manuscript_supporting_axis_partner`, `manuscript_falsified_penalty`).

MONAI, MedSigLIP, TxGemma, ClinicalBERT, and the CBIS-DDSM logreg are not
invoked on this case — different modality. Nothing on the AK page is a
discovery by the arbiter. Everything is verifiable via the linked SHAs.

## 3. Governance invariants (enforced by acceptance tests)

**Always required (T01, part of every visible HTML page):**
- Regulatory strip carries `RESEARCH USE ONLY` and `Not FDA-cleared`.
- A "Human decision boundary" footer that lists the four decisions no
  automated system may make on its own (from
  `quarantine_test_harness.json::required_human_decision_boundary_items`).

**AK page specifically (T02–T06):**
- The mandatory contribution-boundary disclosure verbatim, naming what the
  arbiter did and did not do on this case.
- Three SHAs on-page: manuscript repo, backend repo, canonical
  bundle_sha256. Manuscript SHA linked to the corresponding GitHub tree.
- All 7 anchor p-values render with full decimals (6 primary tests + the
  PARP falsification arm), not rounded.
- PARP falsification is a first-class row in the 6-axis evidence matrix
  with a dedicated `data-block="parp-falsification"` narrative panel. No
  hidden pane.
- Chip discipline: `proxy_co_scientist` chip appears only on the Elo
  audit-sort panel; manuscript-anchored blocks carry `loaded_*` chips.

**NSCLC page specifically (T07):**
- LUNA16 low-confidence (`top_score = 0.042`) is inline, not hidden, with
  the honest explanation that the detected object is out-of-distribution
  for a 3–30 mm nodule detector.

**Everywhere (T08, T09):**
- Zero fake NCT identifiers.
- Zero strings or regex patterns from
  `quarantine_test_harness.json::forbidden_substrings` /
  `forbidden_patterns_regex`. Those are the failing claims that must
  never render anywhere on the deliverable. This README, the mockups, the
  component spec, and both landing pages have all been sweep-tested.

**Reproducibility (T10):**
- Every case has a real ≥3-call transcript on disk at
  `e2e_run/<case>/transcript.jsonl`, all 2xx, with UUID-shaped
  `request_id` fields and the expected `model_state` chip
  (`loaded_luna16_retinanet` for NSCLC, `loaded_ak_bundle` for AK).

## 4. How to reproduce

Two-terminal loop:

```
# Terminal 1 — boot the arbiter
cd oncology_arbiter
uv pip install -e . --system
ONCOLOGY_ARBITER_AUTH_MODE=off \
ONCOLOGY_ARBITER_ALLOW_SERIES_DIR=1 \
ONCOLOGY_ARBITER_ENABLE_LUNA16_RETINANET=1 \
uvicorn oncology_arbiter.api.app:create_app --factory --host 127.0.0.1 --port 8130

# Terminal 2 — replay both transcripts + regenerate receipts + rerun harness
cd tumor_board_frontend
python3 e2e_run/nsclc_case/run_nsclc_e2e.py            # 4 live HTTP calls
python3 e2e_run/ak_ingestion/run_ak_e2e.py             # 3 live HTTP calls
python3 nsclc_luna16/receipts/extract_receipts.py
python3 ak_mbd4_lof/receipts/extract_receipts.py
python3 run_acceptance_tests.py                        # T01–T10
```

`run_acceptance_tests.py` prints a per-test summary and writes
`acceptance_tests.json`. Exit code is nonzero if any test fails.

## 5. Live environment notes

- The MONAI `lung_nodule_ct_detection` bundle is downloaded once to
  `/workspace/monai_bundles/lung_nodule_ct_detection/` (~160 MB, Apache-2.0).
  Metadata: `mAP_IoU_0.10_0.50_step_0.05_MaxDet_100=0.852`, `mAR=0.998`.
  Preprocessing: `hu_clamp=[-1024, 300]`, `target_spacing_mm_zyx=[1.25, 0.703, 0.703]`,
  `roi_size=[192, 192, 80]`, `nms_thresh=0.22`, `score_thresh=0.02`.
- Cold-start on the first `/v1/case/full` call is ~10–30 s (bundle load);
  subsequent calls are ~2–3 s.
- `ONCOLOGY_ARBITER_DEMO_MODE=1` blocks `POST /v1/tumor_board/bundle` with
  403 — do not set it if you want to run the AK case end-to-end.
- FastAPI is factory-only: use `--factory` on uvicorn.
- Package identity: `oncology-arbiter==0.3.0-alpha` installed editable from
  the workspace checkout.
